# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| latest  | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

1. **Do not** open a public issue.
2. Email **adria.sanchez@betterask.erni** with details and reproduction steps.
3. You will receive a response within 48 hours.
4. Do not disclose the vulnerability until a fix is released.

## Security Best Practices

- All dependencies are scanned via Dependabot alerts.
- Pull requests require review before merging.
- Secrets are never committed to the repository.
- CI runs lint and tests on every push and PR.
